Visão Geral

A Document Extractor API é uma solução robusta para extração de texto e imagens de diversos tipos de documentos. A API suporta os seguintes formatos:

Formato Extensão MIME Type Recursos Suportados
PDF .pdf application/pdf Texto, Imagens
Word Document .docx application/vnd.openxmlformats-officedocument.wordprocessingml.document Texto, Tabelas
Word Document (Legacy) .doc application/msword Texto
Excel Spreadsheet .xlsx application/vnd.openxmlformats-officedocument.spreadsheetml.sheet Dados estruturados
Excel Spreadsheet (Legacy) .xls application/vnd.ms-excel Dados estruturados
Texto Simples .txt text/plain Texto
Nota: A API possui um limite de 50MB por arquivo enviado. Para arquivos maiores, considere dividir o documento ou entrar em contato com o suporte.

Autenticação

Atualmente, a API não requer autenticação para uso. Todos os endpoints são públicos e podem ser acessados diretamente.

Aviso: Em um ambiente de produção, recomenda-se implementar autenticação via API Key ou OAuth2 para controlar o acesso e monitorar o uso.

Endpoints

GET /api/health

Verificação de Saúde

Endpoint para verificar se a API está funcionando corretamente.

Resposta

200 OK
{ "status": "healthy", "service": "Document Extractor API", "version": "1.0.0" }
GET /api/supported-types

Tipos de Arquivo Suportados

Retorna a lista de extensões e tipos MIME suportados pela API.

Resposta

200 OK
{ "supported_extensions": ["txt", "pdf", "doc", "docx", "xlsx", "xls"], "mime_types": { "txt": "text/plain", "pdf": "application/pdf", "doc": "application/msword", "docx": "application/vnd.openxmlformats-officedocument.wordprocessingml.document", "xlsx": "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet", "xls": "application/vnd.ms-excel" } }
POST /api/extract

Extração de Documentos

Endpoint principal para extração de texto e imagens de documentos.

Parâmetros

Parâmetro Tipo Obrigatório Descrição
file File Obrigatório Arquivo a ser processado (multipart/form-data)

Exemplo de Requisição

# Exemplo com cURL curl -X POST \ "http://localhost:5000/api/extract" \ -H "Content-Type: multipart/form-data" \ -F "file=@documento.pdf"

Resposta de Sucesso

200 OK
{ "success": true, "data": { "text": "Texto extraído do documento...", "file_info": { "filename": "documento.pdf", "type": "pdf", "mime_type": "application/pdf" }, "pages": 5, "total_images": 3, "images": [ { "page": 1, "index": 1, "format": "png", "data": "base64_encoded_image_data", "width": 800, "height": 600 } ] } }

Resposta de Erro

{ "success": false, "error": "Descrição do erro" }

Estrutura de Resposta por Tipo de Arquivo

PDF

{ "text": "Texto extraído de todas as páginas", "pages": 10, "images": [...], "total_images": 5 }

Word (.docx)

{ "text": "Texto extraído incluindo tabelas", "paragraphs": 25, "tables": 3 }

Excel (.xlsx/.xls)

{ "text": "Dados de todas as planilhas formatados", "sheets": { "Planilha1": { "text": "Dados da planilha...", "rows": 100, "columns": 5, "column_names": ["Nome", "Idade", "Email", "Telefone", "Cidade"] } }, "total_sheets": 3 }

Texto (.txt)

{ "text": "Conteúdo do arquivo de texto", "lines": 50, "characters": 2500, "words": 400 }

Extração via URL

A API suporte extração de documentos diretamente de URLs, baixando e processando automaticamente.

POST /api/extract/url

Descrição

Baixa um documento de uma URL e extrai seu conteúdo. Suporta os mesmos formatos de arquivo que o upload direto.

Parâmetros de Entrada (JSON)

Campo Tipo Obrigatório Descrição
url string Sim URL do documento a ser baixado
max_size_mb number Não Tamanho máximo em MB (padrão: 50)

Exemplo de Requisição

curl -X POST http://localhost:5000/api/extract/url \ -H "Content-Type: application/json" \ -d '{ "url": "https://example.com/document.pdf", "max_size_mb": 20 }'

Resposta de Sucesso

200 OK
{ "success": true, "data": { "text": "Texto extraído do documento...", "file_info": { "filename": "document.pdf", "type": "pdf", "size_mb": 2.5, "processing_time": 1.2 }, "download_info": { "source_url": "https://example.com/document.pdf", "download_successful": true, "content_type": "application/pdf" } } }
POST /api/extract/url/bulk

Descrição

Processa múltiplas URLs em lote. Limite máximo de 5 URLs por requisição.

Parâmetros de Entrada (JSON)

Campo Tipo Obrigatório Descrição
urls array Sim Lista de URLs dos documentos (máx. 5)
max_size_mb number Não Tamanho máximo por arquivo em MB (padrão: 50)

Exemplo de Requisição

{ "urls": [ "https://example.com/doc1.pdf", "https://example.com/doc2.docx", "https://example.com/data.xlsx" ], "max_size_mb": 30 }

Resposta de Sucesso

200 OK
{ "success": true, "results": [ { "index": 0, "url": "https://example.com/doc1.pdf", "filename": "doc1.pdf", "success": true, "data": { /* dados extraídos */ } } ], "errors": [], "summary": { "total_urls": 3, "processed": 3, "failed": 0, "success_rate": 100.0, "total_processing_time": 3.2 } }
🔒 Segurança: A API valida as URLs e bloqueia protocolos não seguros. Apenas URLs HTTP/HTTPS são permitidas. Arquivos muito grandes ou que excedam o timeout são rejeitados.
⚠️ Limitações: O download de URLs está sujeito a timeouts de rede (30 segundos) e limites de tamanho. URLs que requerem autenticação não são suportadas.

Exemplos de Uso

cURL

# Verificar saúde da API curl -X GET http://localhost:5000/api/health # Obter tipos suportados curl -X GET http://localhost:5000/api/supported-types # Extrair dados de um documento curl -X POST http://localhost:5000/api/extract \ -F "file=@documento.pdf"

JavaScript (Fetch API)

// Função para extrair dados de um arquivo async function extractDocument(file) { const formData = new FormData(); formData.append('file', file); try { const response = await fetch('/api/extract', { method: 'POST', body: formData }); const result = await response.json(); if (result.success) { console.log('Texto extraído:', result.data.text); console.log('Informações do arquivo:', result.data.file_info); } else { console.error('Erro:', result.error); } } catch (error) { console.error('Erro de rede:', error); } } // Usar com input de arquivo document.getElementById('fileInput').addEventListener('change', (e) => { if (e.target.files.length > 0) { extractDocument(e.target.files[0]); } });

Python (requests)

import requests # Verificar saúde da API response = requests.get('http://localhost:5000/api/health') print(response.json()) # Extrair dados de um documento with open('documento.pdf', 'rb') as file: files = {'file': file} response = requests.post('http://localhost:5000/api/extract', files=files) if response.status_code == 200: result = response.json() if result['success']: print("Texto extraído:", result['data']['text']) else: print("Erro:", result['error']) else: print("Erro HTTP:", response.status_code)

Códigos de Erro

Código HTTP Erro Descrição
400 Bad Request Nenhum arquivo enviado ou tipo de arquivo não suportado
413 Payload Too Large Arquivo excede o limite de 50MB
500 Internal Server Error Erro interno durante o processamento do arquivo

Exemplos de Mensagens de Erro

// Nenhum arquivo enviado { "error": "Nenhum arquivo foi enviado" } // Tipo de arquivo não suportado { "error": "Tipo de arquivo não suportado", "supported_types": ["txt", "pdf", "doc", "docx", "xlsx", "xls"] } // Erro de processamento { "success": false, "error": "Erro ao extrair texto do PDF: arquivo corrompido" }
